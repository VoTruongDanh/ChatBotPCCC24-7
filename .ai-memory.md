# AI Memory - PCCC Project

## Project Overview
- **Type**: Monorepo with 2 independent modules (Bridge + PCCC)
- **Tech Stack**: Next.js, React, Node.js, Express
- **Architecture**: Module-based config system with centralized .env files

## Recent Enhancements (Latest First)

### 2024-04-18 (v3): Improved Contrast + Auto-focus After Send
**Changes:**
1. **Enhanced Text Contrast for Better Visibility**
   - Light mode text: `#1f2937` (gray-800) → darker, more readable
   - Dark mode text: `#fef3e2` (warm white) → brighter, more visible
   - Increased font-weight from `400` to `500` for input text
   - Dark mode background: darker (`#050301` vs `#0c0704`) for better contrast
   - Light mode background: slightly gray (`#fafafa` vs `#ffffff`) to reduce eye strain
   - Placeholder colors improved:
     - Light: `#9ca3af` (gray-400)
     - Dark: `rgba(254,243,226,0.5)` (50% opacity warm white)

2. **Auto-focus Input After Sending Message**
   - Added `inputRef` to input element
   - After form submit, automatically refocus input with 50ms delay
   - User can continue typing immediately after sending
   - No need to click back into input field

3. **Improved Dark Mode Contrast**
   - Background colors darker: `--bg: #050301`, `--bg2: #0d0805`
   - Border opacity increased: `0.1` and `0.18` (was `0.08` and `0.14`)
   - Text colors brighter: `--text: #fef3e2`, `--text2: rgba(254,243,226,0.75)`
   - Shadow stronger: `0.6` and `0.8` opacity (was `0.4` and `0.6`)
   - Input/bubble backgrounds more opaque: `0.95` (was `0.9`)

4. **Improved Light Mode Contrast**
   - Background slightly gray: `#fafafa` (was pure white `#ffffff`)
   - Text darker: `--text: #0f172a` (slate-900)
   - Border stronger: `0.1` and `0.16` opacity (was `0.08` and `0.14`)
   - Input/bubble more opaque: `0.98` (was `0.95`)

**Impact:**
- ✅ Text much more visible in both themes
- ✅ Input stays focused after sending - better UX for continuous chat
- ✅ Dark mode has true contrast (not washed out)
- ✅ Light mode easier on eyes (not pure white)
- ✅ Font weight 500 makes text pop more

**Tradeoffs:**
- Darker backgrounds in dark mode might feel "heavier"
- Gray background in light mode less "clean" than pure white
- Auto-focus might be unexpected for some users (but improves chat flow)

**Remaining Tech Debt:**
- Save theme preference to localStorage
- Add theme transition animation
- Consider reducing bell animation intensity
- Create pages for: Quy định, Hồ sơ, Liên hệ

### 2024-04-18 (v2): Removed Input Halo + Added Theme Toggle + Bell Animation
**Changes:**
1. **Removed Input Gradient Halo Effect**
   - Completely removed `.pccc-input-halo` element and CSS
   - Removed `z-index` workarounds from input shell
   - Simplified focus state to just border glow (no spinning gradient)
   - Result: Text and UI elements no longer "sink" or get covered

2. **Moved Animation to Send Button (Bell Effect)**
   - Added spinning gradient halo to `.pccc-send-active::before`
   - Added `bellPulse` animation (scale 1 → 1.05 continuously)
   - Added `bellRing` animation on hover (rotate -10° → 10° like a bell)
   - Gradient uses same conic-gradient with `spinHalo` animation
   - Result: Send button has eye-catching notification bell effect

3. **Added Manual Theme Toggle Button**
   - New button in nav with sun/moon icons
   - State managed with `theme` useState ('light' | 'dark')
   - Sets `data-theme` attribute on `<html>` element
   - Auto-detects system preference on mount
   - CSS updated to use `[data-theme="light"]` and `[data-theme="dark"]`
   - Fallback to `prefers-color-scheme` for initial load
   - Smooth icon rotation on hover (20deg)

4. **Fixed Theme Consistency**
   - Input text color now explicitly set per theme:
     - Light: `#111827` (gray-900)
     - Dark: `#f5f0ec` (warm white)
   - Placeholder colors per theme:
     - Light: `#9ca3af` (gray-400)
     - Dark: `rgba(245,240,236,0.4)` (warm white 40%)
   - All CSS variables properly scoped to `[data-theme]`

**Impact:**
- ✅ No more text/UI sinking or being covered by effects
- ✅ Send button has attractive bell/notification animation
- ✅ Users can manually toggle light/dark mode
- ✅ Theme persists and applies immediately
- ✅ Input text always visible in both themes

**Tradeoffs:**
- Lost fancy input focus effect (but gained clarity)
- Theme preference not saved to localStorage (resets on refresh)
- Send button animation might be distracting for some users

**Remaining Tech Debt:**
- Save theme preference to localStorage
- Add theme transition animation (smooth color fade)
- Consider reducing bell animation intensity
- Create pages for: Quy định, Hồ sơ, Liên hệ

## Architecture Decisions

### Module Structure
- **Bridge Module**: `bridge/` - Reusable AI bridge component
  - `bridge/be-bridge/` - Backend API
  - `bridge/ui-bridge/` - Admin UI
  - `bridge/.env` - Single config file
  
- **PCCC Module**: `pccc/` - Main application
  - `pccc/be-main/` - Main backend
  - `pccc/ui/` - User-facing UI
  - `pccc/.env` - Single config file

### Config Management
- Only 2 `.env` files to manage (module-level)
- `scripts/sync-config.mjs` syncs to app-level configs
- Bridge module is portable to other projects
