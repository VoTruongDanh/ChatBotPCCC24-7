### 2026-04-21 (debug): Bridge admin smoke audit - worker pool and browser controls
**Symptom:**
1. Admin `Add worker` could silently corrupt `be-bridge` worker pool.
2. Admin `Show/Hide browser` and browser status could report wrong state or fail even when workers were running.
3. Admin bridge success/error messages were injected into page layout instead of floating independently.

**Root cause:**
1. `handleAdminWorkers(POST)` inserted plain objects into `workerPool` instead of real `WorkerInstance`s. Later, `dispatch()` could pick one of these fake entries and call `sendPromptAndWaitResponse()` on an object that did not implement it.
2. `admin.mjs` relied on `worker.mjs` browser helpers backed by stale module-level `browser/page` globals. `master.mjs` actually drives per-worker `WorkerInstance`s, so admin browser actions were disconnected from the real running workers.
3. `ui-bridge` rendered status banners inline in the main content flow, so notifications pushed the layout down and looked like normal content blocks.

**Fix:**
1. `bridge/be-bridge/src/admin.mjs`
   - `POST /admin/workers` now creates and launches real `WorkerInstance`s.
   - `DELETE /admin/workers` now destroys actual worker instances before removing them from the pool.
   - Partial launch/remove failures are returned as `warning` text instead of being swallowed.
2. `bridge/be-bridge/src/worker.mjs`
   - Added `activeWorkers` registry tied to `WorkerInstance.launch()` / `destroy()`.
   - `isBrowserRunning()`, `showBrowserWindow()`, `hideBrowserWindow()`, `getBrowser()`, `getPage()`, and exported `isGenerating()` now resolve against the live worker set instead of stale globals.
   - Exported fallback `sendPromptAndWaitResponse()` and `startNewTemporaryChat()` now delegate to a real live worker.
3. `bridge/ui-bridge/src/app/page.tsx` and `globals.css`
   - Moved admin success/error feedback to a fixed toast stack in the top-left corner.
   - Added auto-dismiss timers and kept manual close.
4. `bridge/ui-bridge/src/app/TabOverview.tsx`, `TabStatus.tsx`, `TabWorkers.tsx`
   - Rewrote mojibake-corrupted text blocks to valid UTF-8 Vietnamese so the admin UI no longer looks broken.

**Files:**
- `bridge/be-bridge/src/admin.mjs`
- `bridge/be-bridge/src/worker.mjs`
- `bridge/ui-bridge/src/app/page.tsx`
- `bridge/ui-bridge/src/app/globals.css`
- `bridge/ui-bridge/src/app/TabOverview.tsx`
- `bridge/ui-bridge/src/app/TabStatus.tsx`
- `bridge/ui-bridge/src/app/TabWorkers.tsx`

**Verification:**
1. `node --check bridge/be-bridge/src/worker.mjs`

### 2026-04-19 (debug): ECONNREFUSED - Config sync workflow missing
**Symptom:** `be-main` không kết nối được đến `be-bridge`, lỗi `TypeError: fetch failed` với `ECONNREFUSED` khi POST `/api/chat/stream`. User sửa `bridge/.env` nhưng không có hiệu lực.

**Root cause:** Kiến trúc config 2-tier (module-level `bridge/.env` → app-level `bridge/be-bridge/.env`) yêu cầu workflow 3 bước: (1) sửa module `.env`, (2) chạy `npm run config:sync`, (3) restart service. User chỉ làm bước 1, bỏ qua bước 2 và 3 → `be-bridge` vẫn dùng config cũ.

**Fix:** 
1. Chạy `npm run config:sync` để sync từ `bridge/.env` → `bridge/be-bridge/.env`
2. Restart `be-bridge` để đọc config mới

**Files:** `bridge/.env`, `bridge/be-bridge/.env`, `scripts/sync-config.mjs`, `bridge/be-bridge/src/config.mjs`

**Verification:** 
1. Sau restart, kiểm tra log `be-bridge` in đúng port
2. Test health check: `curl http://localhost:1122/health`
3. Test chat từ UI → không còn ECONNREFUSED

**Regression prevention:** 
- Luôn chạy `npm run config:sync` sau khi sửa module `.env`
- Luôn restart services sau khi sync
- Cân nhắc thêm file watcher hoặc pre-start hook tự động sync
- Hoặc đơn giản hóa: bỏ 2-tier, chỉ dùng app-level `.env`

**Tradeoffs:** 2-tier config tốt cho module reusability nhưng tăng complexity và dễ quên sync. Nếu không cần portable module, app-level `.env` đơn giản hơn.

## Architecture Decisions

### Module Structure
- **Bridge Module**: `bridge/` - Reusable AI bridge component
  - `bridge/be-bridge/` - Backend API
  - `bridge/ui-bridge/` - Admin UI
  - `bridge/.env` - Single config file
  
- **PCCC Module**: `pccc/` - Main application
  - `pccc/be-main/` - Main backend
  - `pccc/ui/` - User-facing UI
  - `pccc/.env` - Single config file

### Config Management
- Only 2 `.env` files to manage (module-level)
- `scripts/sync-config.mjs` syncs to app-level configs
- Bridge module is portable to other projects
